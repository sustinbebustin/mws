# decisions/

Lightweight ADRs (Architecture Decision Records) for one-way doors. Capture the *why* behind decisions that are hard to reverse, so future-you (and future agents) don't relitigate them.

Format: `NNNN-short-slug.md`, numbered sequentially.

Template:

```md
# NNNN. <decision title>

Date: YYYY-MM-DD
Status: accepted | superseded by NNNN | deprecated

## Context
What forced the decision. The constraint, the problem, the alternatives considered.

## Decision
What we chose.

## Consequences
What this makes easier, harder, or impossible. What we accept by choosing this.
```

Keep them short. ADRs are signposts, not essays.
