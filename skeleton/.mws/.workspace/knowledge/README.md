# knowledge/

Durable, curated context. Read by agents (and humans) to ground decisions and prevent drift.

| Subfolder | Add here when... |
|---|---|
| `domains/<flow>/` | the content describes a business flow we own. |
| `integrations/<vendor>/` | the content describes a third-party system we call. Raw vendor docs go under `vendor/`. |
| `architecture/` | the content is cross-cutting (codemaps, infra, data/auth models). |
| `engineering/` | the content is generic patterns or external research not tied to this project. |

If a doc is for a single in-flight branch, it belongs in `work/<slug>/`, not here. Promote to `knowledge/` once it's stable.
